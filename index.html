路徑分析

計算兩點之間的最短路徑。結果分兩欄:「狀態」標示正常或繞道(故障重路由),「路徑」標示單一或 ECMP 多路徑分流。
選擇起點和終點後,點擊「計算最短路徑」。

最短路徑矩陣

所有 Router 之間的最短路徑成本一覽表,底色依成本分 4 階。點任一格查看路徑詳情並在圖上高亮。
≤30 — 區域內 31–60 — 跨區域 61–100 — 跨洋 >100 — 多段跨洋 ² = 等成本路徑數(ECMP) = 來回路徑非對稱(斜紋)
SLO 目標 ≤ ms

樞紐度盤點

衡量每條鏈路和每台節點「有多少路徑經過」— 數值越高,越多流量依賴它,故障影響越大。
分 4 級:樞紐(需擴容) / 正常(維持) / 偶爾(可降規) / 純備援(鏈路)·末端(節點)(零過路流量)。

鏈路流量視圖

根據流量需求矩陣,計算每條鏈路承載的 Gbps(等成本路徑平均分攤),對照鏈路容量算出利用率。電路為全雙工、容量單向計,故 Gbps 與利用率取較忙方向(去 / 回擇大)。分 5 階:>100% 過載 · 80–100% 高負載 · 50–80% 中負載 · 0–50% 低負載 · 0% 閒置(無流量,圖上淡灰虛線)。

鏈路成本編輯

三種設定成本的方式:① 表格直接輸入 · ② 點「參考」帶入延遲建議值 · ③ 下方「壅塞最佳化」依流量自動搜尋一組。①② 照延遲、③ 照壅塞,目標不同 → 建議值可能不一樣(正常)。
「成本」是 OSPF 用來挑路徑的權重 — 數字越小,這條鏈路越容易被選為主要路徑。改了成本,圖上標籤會即時更新、路徑自動重算。
兩台路由器直連的鏈路(點對點),可分開設定去程(A→B)和回程(B→A)的成本,兩個方向不一定要一樣。
「參考」欄會依實際來回延遲(RTT)換算出一個建議成本,點一下就能帶入去/回程欄位。若目前成本和建議值差距 > 1,該列參考欄左緣會標上琥珀色條(建議成本數字也轉琥珀),表示這條鏈路的成本可能是人為刻意設定的(例如兩個方向故意設不同值)。
換算公式:建議成本 = round( RTT ÷ (每公里來回延遲 × 90) ),再夾在 5–250 之間(太短不低於 5、太長不超過 250)。目前每公里來回延遲 —,換算後約每 — ms 延遲對應 1 級成本。
建議成本背後的延遲資料有三種來源,可信度由高到低: 🟢 直測 — 直接量到的實際延遲(最準) 🔵 城市參考 — 查公開的「城市對城市」延遲資料庫 ⚪ 模型估計 — 用兩地直線距離推算(最粗略)

壅塞最佳化 (Fortz-Thorup / Tabu)

自動搜尋一組 OSPF 權重,壓低最大鏈路利用率(MLU)= 全網最忙鏈路的利用率;口徑同「鏈路流量」視圖(含 transit、較忙方向)。約束:RTT 下限 + 整數 [5,500]。算完先預覽,確認再套用。需 demand.js + 鏈路 capacity。
與上面「參考」不同:參考照延遲給單條建議,最佳化照流量/容量找整組——所以套用後參考欄常會亮琥珀條(偏離延遲建議),屬正常。
進階選項 — 勾選會額外搜尋,只保留更好的結果

失效模擬

⚠ 設計審計:基於完整拓樸,不受手動故障標記影響
模擬單一元件或整組共同風險群組 (SRLG) 同時失效,檢視網路連通性、流量重分配與容量溢出。SRLG 涵蓋海纜系統、共管線路、機房電力、上游 ISP 等共用風險。

ECMP 備援檢查

⚠ 設計審計:基於完整拓樸,不受手動故障標記影響
當兩點之間有多條等成本路徑(ECMP)時,斷掉其中一條,流量應由同群組的其他路徑接手。本檢查逐一模擬斷線,驗證備援是否留在群組內。
通過斷任一條,群組內接手
失敗斷線後不可達或繞出群組
不適用只有一條路徑,不適用此檢查

非對稱路徑偵測

⚠ 設計審計:基於完整拓樸,不受手動故障標記影響
檢查每對 Router 的去程和回程是否走同樣的路徑、同樣的成本。不對稱時,封包來回走不同路,可能造成延遲不一致或排障困難。
圖上紫框節點 — 非對稱 pair 的兩端 對稱 去回程路徑與成本完全相同 非對稱 去回程成本不同,或路徑集合不同
每組明細以藍色方向標題(A → B)分列兩方向的成本與路徑,並標注差異原因。

Prefix 宣告冗餘

⚠ 設計審計:基於完整拓樸,不受手動故障標記影響
每個網段(Prefix)被幾台 Router 宣告?被 2 台以上宣告 = 有備援;只被 1 台宣告 = 該台故障後網段就斷了。
節點顏色 = 該 Router 宣告的網段中「無備援」的比例:
= 0% — 全 prefix 都有冗餘宣告(理想) 0–33% — 少數 prefix 單一宣告 33–66% — 約一半 prefix 無冗餘 > 66% — 多數 prefix 單一宣告(高風險)

N-1 單點故障排行

⚠ 設計審計:基於完整拓樸,不受手動故障標記影響
逐一模擬每個元件(每條鏈路、每台 Router)單獨故障,找出哪些連線受害最深、哪個元件故障殺傷力最大。 ∞= 故障後不可達。 ×N= 成本變 N 倍。 點任一列在圖上查看影響範圍。
