路徑分析

計算兩點之間的最短路徑。若存在多條相同成本的路徑會全部列出(ECMP)。如果故障導致走了繞道,會標示為「備援模式」。
選擇起點和終點後,點擊「計算最短路徑」。

最短路徑矩陣

所有 Router 之間的最短路徑成本一覽表。底色越紅 = 成本越高; 右上角數字 = 等成本路徑數(ECMP);右下 * = 來回路徑不對稱。 點任一格查看路徑詳情並在圖上高亮。

樞紐度盤點

衡量每條鏈路和每台節點「有多少路徑經過」— 數值越高,越多流量依賴它,故障影響越大。
分 4 級:樞紐(需擴容) / 正常(維持) / 偶爾(可降規) / 純備援(可裁併)。

邊流量視圖

根據流量需求矩陣,計算每條鏈路實際承載多少 Gbps(等成本路徑平均分攤),並對照鏈路容量算出利用率。分 4 階:>100% 過載 · 80–100% 高負載 · 50–80% 中負載 · <50% 低負載。

鏈路成本編輯

修改鏈路成本後,圖上標籤即時更新,路徑自動重算。點對點鏈路可分別設定去程和回程的成本。

節點失效模擬

⚠ 設計審計:基於完整拓樸,不受手動故障標記影響
假設一台 Router 完全故障:網路是否仍然連通?流量會怎麼重新分配?哪些鏈路負載會增加?是否超過容量?

ECMP 備援檢查

⚠ 設計審計:基於完整拓樸,不受手動故障標記影響
當兩點之間有多條等成本路徑(ECMP)時,斷掉其中一條,流量應由同群組的其他路徑接手。本檢查逐一模擬斷線,驗證備援是否留在群組內。
通過斷任一條,群組內接手
失敗斷線後不可達或繞出群組
不適用只有一條路徑,不適用此檢查

非對稱路徑偵測

⚠ 設計審計:基於完整拓樸,不受手動故障標記影響
檢查每對 Router 的去程和回程是否走同樣的路徑、同樣的成本。不對稱時,封包來回走不同路,可能造成延遲不一致或排障困難。

Prefix 宣告冗餘

⚠ 設計審計:基於完整拓樸,不受手動故障標記影響
每個網段(Prefix)被幾台 Router 宣告?被 2 台以上宣告 = 有備援;只被 1 台宣告 = 該台故障後網段就斷了。
節點顏色 = 該 Router 宣告的網段中「無備援」的比例:
= 0% — 全 prefix 都有冗餘宣告(理想) 0–33% — 少數 prefix 單一宣告 33–66% — 約一半 prefix 無冗餘 > 66% — 多數 prefix 單一宣告(高風險)

N-1 單點故障排行

⚠ 設計審計:基於完整拓樸,不受手動故障標記影響
逐一模擬每個元件(每條鏈路、每台 Router)單獨故障,找出哪些連線受害最深、哪個元件故障殺傷力最大。 ∞= 故障後不可達。 ×N= 成本變 N 倍。 點任一列在圖上查看影響範圍。
